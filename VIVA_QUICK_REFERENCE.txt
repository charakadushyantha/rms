┌──────────────────────────────────────────────────────────────────────────────┐
│ COMMON VIVA QUESTIONS & ANSWERS                                              │
└──────────────────────────────────────────────────────────────────────────────┘

Q: Is this using dummy data?
A: No, all data is real from MySQL database. I can add a candidate now to prove it.

Q: What framework did you use?
A: CodeIgniter 3 (PHP MVC framework) with Bootstrap 5 for frontend.

Q: How do you prevent SQL injection?
A: Using CodeIgniter's Active Record/Query Builder which sanitizes inputs.

Q: What's the database structure?
A: Normalized relational database with 8+ tables including candidate_details,
   calendar_events, users, questions_bank, and job_postings.

Q: How does interview scheduling work?
A: Admin selects candidate, interviewer, date/time, and type. System stores
   in calendar_events table and displays on calendar with filtering options.

Q: Can you show the code?
A: Yes, controllers are in application/controllers/, views in
   application/views/, and models in application/models/.

Q: What happens when you delete an interview?
A: System removes record from calendar_events table and redirects to calendar
   with success message. Can be demonstrated live.

Q: How do you handle authentication?
A: Session-based authentication. Users login, session stores credentials,
   each page checks session before loading. Logout destroys session.
