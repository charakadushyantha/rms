🚀 QUICK SETUP (3 STEPS):

   1. TEST THE CONFIGURATION
      → Open: http://localhost/your-project/test_central_config.php
      → Verify all tests pass

   2. MIGRATE YOUR SCRIPTS
      → Open: http://localhost/your-project/migrate_to_central_config.php
      → This updates all PHP files automatically
      → Backups are created in backups_db_migration/

   3. CREATE .ENV FILE (Optional - for production)
      → Copy .env.example to .env
      → Edit with your production credentials

✅ BENEFITS:

   ✓ Change database credentials in ONE place
   ✓ All scripts update automatically
   ✓ Support for dev/staging/production environments
   ✓ Secure credential management
   ✓ Better error handling
   ✓ No more hardcoded passwords

📝 USAGE IN YOUR SCRIPTS:

   OLD WAY (Before):
   ─────────────────────────────────────────────────────────
   $host = 'localhost';
   $username = 'root';
   $password = '';
   $database = 'rmsdb';
   $conn = new mysqli($host, $username, $password, $database);
   ─────────────────────────────────────────────────────────

   NEW WAY (After):
   ─────────────────────────────────────────────────────────
   require_once __DIR__ . '/config/database.php';
   $conn = getDatabaseConnection();
   ─────────────────────────────────────────────────────────

🔧 AVAILABLE FUNCTIONS:

   getDatabaseConnection()    → Get mysqli connection
   getDatabaseConfig()        → Get config array
   testDatabaseConnection()   → Test if connection works

🔐 SECURITY:

   • .env file is in .gitignore (credentials not in version control)
   • Use environment variables in production
   • Never commit .env file to git
